# 파이썬 & Git 기초 마스터 퀴즈 게임

## 1. 프로젝트 개요
이 프로젝트는 파이썬(Python)의 핵심 문법과 Git 버전 관리 시스템의 기초를 복습할 수 있는 콘솔 기반의 인터랙티브 퀴즈 게임입니다. 단순히 코드를 따라 치는 것을 넘어, 객체 지향 프로그래밍(OOP) 패러다임을 적용하여 클래스 단위로 역할을 분리하고, JSON 파일 입출력을 통해 데이터를 영구적으로 저장하는 '데이터 영속성'을 직접 구현했습니다. 또한, 전체 개발 과정을 Git으로 관리하며 브랜치 생성, 병합(Merge) 등의 실전 협업 워크플로우를 경험하는 것을 목표로 제작되었습니다.

## 2. 퀴즈 주제 선정 이유
본격적인 소프트웨어 개발을 심도 있게 학습하는 3학년 과정에 접어들면서, 새로운 기술을 배우는 것만큼이나 개발의 가장 기본이 되는 핵심 도구들을 완벽하게 다루는 것이 중요하다는 것을 체감했습니다. 파이썬의 자료형, 제어문, 그리고 Git의 원격 저장소 관리 명령어 등은 매일 사용하는 필수 지식이므로, 이를 퀴즈 형태로 스스로 묻고 답하며 확실하게 내재화하기 위해 이 주제를 선정했습니다. 내가 직접 구조를 짠 프로그램으로 나의 성장을 돕는 도구를 만들었다는 점에서 큰 의미가 있습니다.

## 3. 주요 기능 목록

### 🎯 기본 기능
* **인터랙티브 메뉴 인터페이스**: 직관적인 번호 선택을 통해 프로그램의 모든 기능(풀기, 추가, 목록, 확인, 종료)에 쉽게 접근할 수 있습니다.
* **퀴즈 풀기 및 채점**: 등록된 퀴즈를 풀고, 입력한 답과 정답을 대조하여 즉시 채점 결과를 제공합니다.
* **새로운 퀴즈 동적 추가**: 프로그램 실행 중에도 사용자 입력을 받아 새로운 문제, 4개의 선택지, 정답 및 힌트를 자유롭게 추가할 수 있습니다.
* **전체 퀴즈 목록 조회**: 현재 시스템에 등록된 모든 퀴즈의 문제와 정답을 한눈에 확인할 수 있습니다.
* **점수 관리 및 데이터 영속성 (`state.json`)**: 게임 종료 후에도 새로 추가한 퀴즈와 역대 최고 점수 데이터가 `state.json` 파일에 안전하게 보존되며, 다음 실행 시 자동으로 로드됩니다.
* **강건한 예외 처리**: 사용자의 빈 입력, 문자 입력, 범위를 벗어난 숫자 등 비정상적인 입력이나 강제 종료(Ctrl+C) 시에도 프로그램이 다운되지 않고 안전하게 데이터를 저장합니다.

### 🚀 보너스 기능 (선택 구현 사항 적용 완료)
* **문제 랜덤 출제 및 개수 선택**: 매번 같은 순서로 문제를 푸는 지루함을 없애기 위해 퀴즈를 무작위로 섞어 출제하며, 사용자가 풀고 싶은 문제의 개수를 직접 지정할 수 있습니다.
* **힌트 시스템 및 페널티**: 퀴즈 풀이 중 'h'를 입력하여 힌트를 열람할 수 있으며, 힌트 사용 시 해당 문제의 획득 점수가 절반으로 감소하는 게임적 요소를 적용했습니다.
* **퀴즈 삭제 기능**: 불필요하거나 잘못 입력된 퀴즈를 번호 선택으로 간편하게 삭제하고 저장소에 즉각 반영합니다.
* **상세 플레이 히스토리 기록**: 단순한 최고 점수뿐만 아니라, 플레이 날짜, 푼 문제 수, 획득 점수 등 최근 5회의 세부 플레이 기록을 누적하여 저장하고 조회할 수 있습니다.

## 6. 데이터 관리 및 파일 설명 (`state.json`)

### 파일 경로 및 역할
* **경로**: 프로젝트 루트 디렉터리 (`./state.json`)
* **역할**: 프로그램을 종료하거나 시스템을 재시작해도 데이터가 유지될 수 있도록 모든 상태를 UTF-8 인코딩의 JSON 형식으로 영구 저장합니다. 파일이 존재하지 않거나 손상된 경우 프로그램은 자동으로 기본 데이터를 생성하여 복구합니다.

### 데이터 구조 (스키마)
데이터 파일은 아래와 같은 키-값 구조로 구성됩니다.

* **`best_score`** (Number): 현재까지 달성한 최고 점수 기록입니다.
* **`history`** (Array): 최근 플레이한 게임 기록들의 리스트입니다.
    * `date`: 플레이 시각
    * `played`: 풀이한 문제 수
    * `score`: 획득 점수
* **`quizzes`** (Array): 시스템에 등록된 모든 퀴즈 객체들의 리스트입니다.
    * `question`: 퀴즈 문제 텍스트
    * `choices`: 4개의 선택지가 담긴 리스트
    * `answer`: 정답 번호 (1~4)
    * `hint`: 문제 풀이에 도움을 주는 힌트 내용

**구성 예시:**
```json
{
    "best_score": 100,
    "history": [
        {
            "date": "2026-04-22 19:30:00",
            "played": 5,
            "score": 50
        }
    ],
    "quizzes": [
        {
            "question": "Git에서 변경 사항을 원격 저장소로 올리는 명령어는?",
            "choices": ["git pull", "git push", "git commit", "git clone"],
            "answer": 2,
            "hint": "밀어넣는다(push)는 의미를 가집니다."
        }
    ]
}
```

## Git 핵심 개념 및 워크플로우 가이드

### 1. Git이란 무엇이며, 왜 필요한가?
Git은 소스 코드의 변경 이력을 추적하고 여러 사람과 안전하게 협업할 수 있도록 돕는 **분산 버전 관리 시스템(VCS)**입니다.
* **버전 관리:** `main_v1.py`, `main_final.py`처럼 파일을 계속 복사할 필요 없이, 하나의 파일에서 과거의 모든 수정 내역을 기록하고 필요 시 특정 시점으로 코드를 되돌릴 수 있습니다.
* **안전한 실험:** 원본 코드를 보존한 상태로 새로운 기능을 안전하게 개발하고 테스트할 수 있습니다.
* **협업:** 여러 개발자가 동시에 같은 프로젝트를 작업하고, 코드 충돌을 최소화하며 병합할 수 있습니다.

### 2. 필수 Git 명령어 실전 예제

* **`git init` (저장소 초기화)**
  * 현재 디렉터리를 Git이 관리하는 로컬 저장소로 설정합니다. (눈에 보이지 않는 `.git` 폴더가 생성됨)
  ```bash
  git init
  ```

* **`git clone` (저장소 복제)**
  * 원격 저장소(GitHub)의 전체 코드와 변경 이력을 로컬 환경으로 복제하여 다운로드합니다.
  ```bash
  git clone [https://github.com/username/quiz-game.git](https://github.com/username/quiz-game.git)
  ```

* **`git add` (스테이징)**
  * 작업하여 내용이 변경된 파일 중, 다음 커밋(저장)에 포함할 파일들을 스테이징 영역(Staging Area)에 등록합니다.
  ```bash
  git add game.py       # 특정 파일만 추가
  git add .             # 현재 디렉터리의 모든 변경사항을 한 번에 추가
  ```

* **`git commit` (변경사항 확정)**
  * 스테이징된 파일들을 하나의 의미 있는 작업 단위로 묶어 로컬 저장소에 영구 기록(버전 생성)합니다.
  ```bash
  git commit -m "Feat: 퀴즈 랜덤 출제 기능 추가"
  ```

* **`git push` (원격 저장소 업로드)**
  * 로컬 저장소에 커밋(기록)된 내역들을 GitHub 등 원격 저장소로 전송하여 공유합니다.
  ```bash
  git push origin main
  ```

* **`git pull` (원격 저장소 동기화)**
  * 원격 저장소에 있는 최신 변경 사항을 로컬 저장소로 가져와 코드를 최신 상태로 병합(Merge)합니다. 새로운 작업을 시작하기 전 동기화를 위해 사용합니다.
  ```bash
  git pull origin main
  ```

* **`git checkout` (브랜치 이동 및 복원)**
  * 다른 브랜치로 이동하거나, 과거의 특정 커밋 상태로 작업 디렉터리를 변경할 때 사용합니다.
  ```bash
  git checkout main             # main 브랜치로 이동
  git checkout -b feature-hint  # feature-hint 브랜치를 새로 생성함과 동시에 이동
  ```

### 3. 브랜치(Branch) 생성 및 병합(Merge) 워크플로우
안전한 개발을 위해 운영 중인 기본 코드(`main`)를 직접 수정하지 않고, 별도의 작업 공간(브랜치)을 만들어 기능 개발 후 원본에 합치는 과정입니다.

**[작업 시나리오: 새로운 퀴즈 삭제 기능 만들기]**

1. 새로운 기능을 안전하게 개발할 독립적인 브랜치를 생성하고 이동합니다.
```bash
git checkout -b feature/delete-quiz
```

2. 기능 개발(예: `game.py` 수정) 완료 후, 해당 브랜치에서 변경사항을 확정합니다.
```bash
git add game.py
git commit -m "Feat: 퀴즈 삭제 메서드 구현 및 예외 처리"
```

3. 병합을 위해 다시 원본 코드가 있는 `main` 브랜치로 돌아옵니다.
```bash
git checkout main
```

4. 작업이 완료된 브랜치의 내용을 `main` 브랜치로 병합(Merge)합니다.
```bash
git merge feature/delete-quiz
```

5. 최종 병합된 결과물을 원격 저장소에 업로드합니다.
```bash
git push origin main
```

## 8. 주요 클래스 및 함수 참조 구조 (Call Graph)
본 프로젝트는 역할별로 3개의 파일(`main.py`, `game.py`, `quiz.py`)로 책임을 분리하여 설계되었습니다.

### 📍 `main.py` (프로그램 진입점 및 UI 제어)
사용자와의 상호작용을 담당하며, 입력값을 검증한 뒤 `QuizGame` 객체에 명령을 위임합니다.
* **`main()`**
  * `QuizGame()` 객체를 생성하여 게임을 초기화합니다.
  * 입력 검증을 위해 내부 헬퍼 함수인 `get_valid_int()`를 지속적으로 호출합니다.
  * 사용자의 메뉴 선택에 따라 `QuizGame` 객체의 주요 메서드(`play_game`, `add_quiz`, `get_quiz_list`, `delete_quiz`, `get_score_info`, `save_data`)를 호출하여 실행 흐름을 제어합니다.
* **`get_valid_int()`**
  * 외부 의존성 없이 독립적으로 동작하며, 입력 공백 제거, 형변환, 예외 처리(빈 입력, 문자 입력, 범위 이탈)를 수행하여 안전한 정수만 반환합니다.

### 🎮 `game.py` - `QuizGame` 클래스 (비즈니스 로직 및 상태 관리)
게임의 전반적인 상태(최고 점수, 퀴즈 목록)를 관리하고 파일 입출력을 통제합니다.
* **`__init__()`**
  * 객체 생성 시 내부적으로 `load_data()`를 즉시 참조하여 기존 데이터를 불러옵니다.
* **`load_data()`**
  * `state.json`을 읽고, `quiz.py`의 `Quiz.from_dict()` 팩토리 메서드를 참조하여 JSON 데이터를 `Quiz` 객체 리스트로 역직렬화합니다. 
  * 파일이 없거나 비어있으면 내부 메서드인 `_init_default_data()`를 호출합니다.
* **`_init_default_data()`, `add_quiz()`, `delete_quiz()`**
  * 데이터에 변동이 생기는 이 3개의 메서드는 작업 완료 직후 반드시 내부 메서드인 `save_data()`를 참조하여 변경 사항을 파일에 즉각 동기화합니다.
* **`save_data()`**
  * `quiz.py`의 `Quiz.to_dict()` 메서드를 참조하여 `Quiz` 객체들을 JSON 형태로 직렬화하여 저장합니다.
* **`play_game()`**
  * 출제 과정에서 개별 퀴즈의 정보를 다루기 위해 `quiz.py`의 `display()`, `get_hint()`, `check_answer()` 메서드를 차례대로 참조합니다. 게임 종료 시 기록 갱신을 위해 `save_data()`를 호출합니다.

### 📝 `quiz.py` - `Quiz` 클래스 (데이터 모델)
개별 퀴즈의 속성과 행위를 정의한 순수 데이터 모델로, 다른 파일의 함수를 참조하지 않고 수동적으로 호출됩니다.
* **`check_answer(user_answer)`**: 입력받은 답과 인스턴스의 정답을 비교하여 Bool 값을 반환합니다.
* **`display()`, `get_hint()`**: 화면 출력용 포맷팅을 담당합니다.
* **`to_dict()`, `from_dict()`**: 파일 입출력을 돕는 직렬화/역직렬화 전용 메서드입니다.
